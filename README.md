# 蓝牙键盘 Android 应用

## 项目简介

这是一个让 Android 手机充当电脑/平板蓝牙键盘的应用。通过蓝牙 HID 协议模拟真实键盘，接收端无需额外安装软件。应用具有现代、简洁的用户界面，操作直观。

## 功能特性

### 1. 蓝牙配对与连接管理
- 界面顶部设置蓝牙配对按钮
- 点击按钮启动蓝牙设备扫描，列出可连接设备
- 用户选择目标设备（电脑或平板）进行配对和连接
- 连接成功后，配对按钮自动收起/隐藏
- 在输入框左上方显示当前已连接设备名称

### 2. 文字输入与传输
- 界面中央有输入框
- 点击输入框唤起手机系统当前使用的默认输入法
- 保留用户原始输入习惯，不做特殊键盘适配
- 输入的文字实时显示在手机应用的输入框内
- 同时通过蓝牙 HID 发送到已连接的电脑/平板

### 3. 发送（Enter）功能
- 在"设置"中提供开关选项："将键盘的换行按钮替换为发送按钮"
- 开启后，手机输入法键盘上的换行键功能变更为"发送"
- 点击发送按钮，接收设备会收到 Enter（回车）事件

### 4. 个性化界面
- **整体风格**：现代、圆润、灵动，控件和布局采用较大圆角和柔和过渡
- **默认背景**：渐变深色背景（从深紫到深蓝）
- **用户自定义背景**：可以在设置中上传本地图片作为应用背景
- **输入框样式**：毛玻璃效果（模糊透明），与背景层次分明

## 技术实现

### 开发环境
- **语言**：Kotlin
- **最小 SDK**：API 28 (Android 9.0+)
- **目标 SDK**：API 31 (Android 12)
- **开发工具**：Android Studio

### 核心技术
1. **蓝牙 HID 协议**：使用 `BluetoothHidDevice` API 实现键盘仿真
2. **权限管理**：处理蓝牙、位置、存储等运行时权限
3. **UI 设计**：使用 Material Design 组件，实现毛玻璃效果
4. **数据持久化**：SharedPreferences 存储用户设置
5. **图片处理**：支持从相册或相机选择自定义背景

## 快速开始（开发者）

### 1. 环境准备
- 安装 Android Studio（建议最新版本）
- 确保 Android SDK 28+ 已安装
- 准备一台 Android 9.0+ 的设备或模拟器（需要支持蓝牙 HID）

### 2. 导入项目
1. 打开 Android Studio
2. 选择 "Open an existing project"
3. 导航到项目根目录，选择打开
4. 等待 Gradle 同步完成

### 3. 编译运行
1. 连接 Android 设备或启动模拟器
2. 点击 Android Studio 工具栏的 "Run" 按钮
3. 选择目标设备
4. 等待应用安装和启动

### 4. 使用步骤
1. **权限授予**：首次启动需要授予蓝牙、位置等权限
2. **配对设备**：点击顶部蓝牙图标，选择要连接的电脑/平板
3. **开始输入**：在输入框中点击，使用系统输入法输入文字
4. **发送内容**：点击发送按钮或回车键发送文本
5. **自定义设置**：点击设置图标可更改背景和键盘设置

## 权限说明

应用需要以下权限：

### 必需权限
- `BLUETOOTH` - 基础蓝牙功能
- `BLUETOOTH_ADMIN` - 蓝牙设备管理
- `BLUETOOTH_CONNECT` - Android 12+ 蓝牙连接
- `BLUETOOTH_SCAN` - Android 12+ 蓝牙扫描
- `ACCESS_FINE_LOCATION` - 蓝牙设备发现（Android 6.0+）

### 可选权限
- `READ_EXTERNAL_STORAGE` - 选择自定义背景图片
- `WRITE_EXTERNAL_STORAGE` - 保存自定义背景
- `CAMERA` - 拍照作为背景

## 注意事项

### 蓝牙 HID 限制
- 需要 Android 9.0+ 设备支持
- 接收设备需要支持蓝牙键盘输入
- 部分旧版本 Android 可能无法正常使用 HID 功能

### 兼容性说明
1. **Android 版本**：最低要求 Android 9.0 (API 28)
2. **蓝牙支持**：设备需要支持蓝牙 4.0+
3. **接收设备**：Windows、macOS、Linux、Android、iOS 等支持蓝牙键盘的设备
4. **权限处理**：Android 6.0+ 需要动态请求位置权限

### 开发注意事项
1. 测试时确保目标设备蓝牙已开启
2. 首次配对可能需要确认配对码
3. 某些设备可能需要额外的权限配置
4. 模拟器可能无法正常测试蓝牙功能

## 已知问题和解决方案

### 1. 蓝牙连接不稳定
- **问题**：连接频繁断开
- **解决**：确保设备在有效范围内，避免信号干扰

### 2. 输入延迟
- **问题**：文字传输有明显延迟
- **解决**：减少同时连接的蓝牙设备数量，确保网络稳定

### 3. 权限拒绝
- **问题**：应用无法获取蓝牙权限
- **解决**：检查系统权限设置，重新授予权限

### 4. 背景图片不显示
- **问题**：自定义背景无法加载
- **解决**：检查存储权限，确保图片格式正确
